material_methods.py¶

Material functions and custom constitutive law classes for SLS/ULS concrete and CFRP reinforcement, used in the structural benchmarking suite.

Author: Elliot Melcer

class CrackingConcreteLawEC(strains: ndarray, stresses: ndarray, eps_cu1: float, eps_c1: float, **kwargs)[Quellcode]¶

Bases: UserDefined

Concrete constitutive law that handles tensile cracking in moment-curvature analysis.

Overrides get_ultimate_strain() of UserDefined so that the tensile strain limit is set to 1e6 (effectively infinite). This prevents the concrete cracking inside get_balanced_failure_strain() from producing the governing curvature; chi_ultimate is therefore controlled by the two failure modes reinforcement fracture vs. concrete crushing.

get_ultimate_strain(yielding: bool = False)[Quellcode]¶

Return the compressive and tensile strain limits.

The concrete tensile limit is fixed at 1e6 to ensure it is never the governing strain in get_balanced_failure_strain(). The compressive limit depends on whether the yielding state is requested.

Parameter:

yielding (bool, optional) – If True, return the strain at peak stress (eps_c1) as the compressive limit; otherwise return the ultimate compressive strain (eps_cu1). Default is False.

Rückgabe:

(compressive_limit, 1e6) where compressive_limit is either eps_c1 or eps_cu1 depending on yielding [-].

create_sls_concrete_EC(conc: Concrete | float | int, constitutive_law: str) → Concrete[Quellcode]¶

Create an SLS Concrete object with constitutive law according to [8].

Parameter:

  • conc (Concrete or float or int) – Either a Concrete instance or a characteristic cylinder strength fck [N/mm²].

  • constitutive_law (str) –

    Keyword selecting the tension/compression law pair. Case- and separator-insensitive. Available options:

    • "NONE_PARABOLIC" — no tensile capacity; Sargin compression.

    • "FCTM_PARABOLIC" — linear elastic tension up to fctm; Sargin compression.

    • "TENSTIFF_PARABOLIC" — linear elastic tension up to fctm followed by tension stiffening per Nayal and Rasheed [4]; Sargin compression.

    • "ELASTIC_ELASTIC" — fully elastic in both tension and compression (no cracking or crushing).

Rückgabe:

A Concrete instance configured for SLS analysis.

Rückgabetyp:

Concrete

Verursacht:

ValueError – If constitutive_law does not match any of the available keywords.

fctm_parabolic_law_EC(concrete: Concrete | float | int, n_c: int = 80, n_t: int = 20) → CrackingConcreteLawEC[Quellcode]¶

Build a constitutive law with linear tension and Sargin compression in accordance with Eurocode 2 [8].

The tension branch is linear elastic from zero to eps_ctm (the mean cracking strain); above eps_ctm the stress drops to zero. The compression branch follows the Sargin parabola from eps_cu1 to zero.

Parameter:

  • concrete (Concrete or float or int) – Either a Concrete instance or a characteristic cylinder strength fck [N/mm²].

  • n_c (int, optional) – Number of discretization points on the compression branch. Default is 80.

  • n_t (int, optional) – Number of discretization points on the linear tension branch. Default is 20.

Rückgabe:

A CrackingConcreteLawEC instance with the assembled strain/stress arrays.

Rückgabetyp:

CrackingConcreteLawEC

tenstiff_parabolic_law_EC(concrete: Concrete | float | int, n_c: int = 80, n_t: int = 20) → TensionStiffeningConcreteLawEC[Quellcode]¶

Build a tension-stiffening constitutive law with Sargin compression according to Eurocode 2 [8] + Nayal and Rasheed [4].

The compression branch follows the Sargin parabola. The tension branch is linear elastic up to fctm, then descends through the tension-stiffening model defined by the fixed parameters P_t = 0.8, R_t = 0.45, S_t = 4, F_t = 10 from [4]. This is an assumption that is yet to be verified experimentally for CRFP concrete sections

Parameter:

  • concrete (Concrete or float or int) – Either a Concrete instance or a characteristic cylinder strength fck [N/mm²].

  • n_c (int, optional) – Number of discretization points on the compression branch. Default is 80.

  • n_t (int, optional) – Number of discretization points on the linear tension branch. Default is 20.

Rückgabe:

A TensionStiffeningConcreteLawEC instance with the assembled strain/stress arrays.

Rückgabetyp:

TensionStiffeningConcreteLawEC

get_cfrp_reinforcement_from_registry(mat_id: str, materials: dict, prestress_percent: float = 0.0, gamma_s: float = 1.3)[Quellcode]¶

Create a CFRP reinforcement object from a materials registry.

CFRP is treated as a brittle-elastic material (no yielding plateau); the constitutive law is therefore a linear Elastic law truncated at the ultimate strain epsuk.

Parameter:

  • mat_id (str) – Material identifier key in the registry (e.g. "solidian GRID Q142/142-CCE-25").

  • materials (dict) – Materials registry dictionary, typically loaded from a CSV file. Each entry must contain the keys type, f_yk, f_tk, E_tex, eps_u, and weight.

  • prestress_percent (float, optional) – Prestress level as a percentage of the ultimate strain (0–100). 0.0 means no prestress; 50.0 means prestressed to 50 % of epsuk. Default is 0.0 [%].

  • gamma_s (float, optional) – Partial safety factor for reinforcement according to [1]. Default is 1.3 [-]

Rückgabe:

A Reinforcement instance configured with the CFRP elastic constitutive law and the computed initial strain.

Rückgabetyp:

Reinforcement

Verursacht:

  • KeyError – If mat_id is not found in materials.

  • ValueError – If the material type is not "reinforcement", or if any required property (f_yk, f_tk, E_tex, eps_u, weight) is None.
